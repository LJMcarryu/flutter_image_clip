# 故障排查

## HEIC / HEIF 无法解码

现象：

- 抛出 `ImageClipUnsupportedFormatException`。
- Android 或 iOS 上相册图片可以预览，但进入 Dart pipeline 后失败。

处理：

- 移动端优先使用 `ImageClipPlatformDecodeAdapter`。
- 如果相册 SDK 已经转为 JPEG/PNG，确认传入的是转换后的 bytes。
- Android HEIC 支持依赖系统解码器，旧系统或部分厂商 ROM 可能仍不支持。

```dart
final processor = ImageProcessor(
  decodeAdapter: const ImageClipPlatformDecodeAdapter(),
);
```

## 大图导致内存升高

现象：

- 处理长截图或相机原图时卡顿、OOM 或系统杀进程。
- benchmark 的 RSS delta 接近 baseline 上限。

处理：

- 使用 `ImageClipDecodeSettings.preview` 限制编辑器预览长边。
- 设置 `ImageClipProcessingSettings.maxInputPixels` 和 `maxOutputPixels`。
- 不需要交互 UI 时优先使用 `processFile`，避免业务层提前读入大图。

## EXIF 方向不符合预期

现象：

- 相机拍摄照片裁剪后方向错误。
- 预览方向与保存结果不一致。

处理：

- 使用 `decodeBytes` / `decodeFile` 入口，让库在解码时烘焙 EXIF orientation。
- 不要在业务层对同一张图重复旋转 EXIF 和像素。
- 如果复现只发生在平台 adapter，请补充设备型号、系统版本和原图样本。

## CI golden 在远端失败

现象：

- 本地 golden 通过，GitHub Actions 上 `matchesGoldenFile` 出现小比例像素 diff。

处理：

- 对 full-page editor golden 使用小容差 comparator。
- 关键布局用尺寸、位置和语义测试兜底。
- 不要把跨平台抗锯齿差异作为 release blocker。

## iOS Privacy Manifest 缺失

现象：

- Xcode 或 App Store Connect 提示依赖缺少 privacy manifest。

处理：

- 确认发布包包含 `ios/Resources/PrivacyInfo.xcprivacy`。
- 确认 `ios/flutter_image_clip.podspec` 中声明了 `s.resource_bundles`。
- 当前库不采集数据、不追踪用户，也不声明 required reason API。

## pub.dev 自动发布失败

现象：

- `Publish to pub.dev` job 在认证阶段失败。

处理：

- 确认 pub.dev Admin 页面已启用 GitHub Actions 自动发布。
- Repository 必须是 `LJMcarryu/flutter_image_clip`。
- Tag pattern 必须是 `v{{version}}`。
- 如果 pub.dev 要求 environment，GitHub workflow 和 repository environment 都应使用 `pub.dev`。
