# 迁移指南

## 从 0.7.0 升级到 0.7.1

`0.7.1` 是稳定性和发布流程修复版本，没有破坏性 API 变化。

建议执行：

```sh
flutter pub upgrade flutter_image_clip
```

然后跑业务侧图片裁剪回归：

- JPEG / PNG 常规裁剪。
- HEIC / HEIF 平台解码。
- 大图预览和保存。
- 自定义 `ImageClipEditorTheme`。

## 从 0.6.x 升级到 0.7.x

重点变化：

- 包版本提升到 `0.7.x`，最低兼容声明为 Dart `>=3.10.0 <4.0.0`、Flutter `>=3.38.1`。
- `ImageClipPlatformDecodeAdapter` 的平台异常会映射为 Dart typed exception。
- `ImageClipEditorTheme` 增加顶部栏、底部栏、保存按钮和比例选项间距等布局 token。
- 发布包不再包含内部测试、benchmark 和 tool 文件。
- iOS plugin 包含 `PrivacyInfo.xcprivacy`。

推荐检查：

- 业务是否直接 import 了 `src/` 路径。如果有，改为 `package:flutter_image_clip/flutter_image_clip.dart`。
- 自定义主题是否依赖旧的硬编码布局。如果有，改为使用 `ImageClipEditorTheme` 新增 token。
- 异常捕获是否只捕获了 `ImageClipDecodeException`。如果需要区分平台前置错误，补充捕获 `ImageClipPlatformException`。

## 从 0.5.x 升级到 0.6.x

重点变化：

- 包升级为 Android/iOS Flutter plugin。
- 新增平台 sampled decode 和 HEIC/HEIF 归一化入口。
- 集成测试迁移到 `example/integration_test`。
- 编辑器默认继承业务 App 字体。

推荐检查：

- Android/iOS 工程重新执行 `flutter pub get`。
- 如果业务使用自定义 native adapter，确认实现的是 `supportsDecode`。
- 如果依赖旧示例结构，改看 `example/lib/main.dart` 和 `guides/接入配方.md`。

## 公开 API 策略

- 只承诺 `package:flutter_image_clip/flutter_image_clip.dart` 和 `package:flutter_image_clip/image_processing/image_processor.dart` 导出的 API。
- `src/` 下实现细节可能在 minor 或 patch 版本中调整。
- 公开 API 变化必须更新 `tool/api_snapshot.json`。
- 标记为 deprecated 的 API 至少保留一个 minor 周期。
